80  // 200 status code is by far the most common returned.
81  // It means, simply, that the request was received and understood

84  // 201 status code indicates that a request was successful
85  // and as a result, a resource has been created (for example a new page).
86  "202 Accepted",
87  // 202 indicates that server has received and understood the request,
88  // and that it has been accepted for processing, although it may not
89  // be processed immediately.
90  "204 No Content",
91  // 204 means that the request was received and understood, but that
92  // there is no need to send any data back.

95  "301 Moved Permanently",
96  // 301 tells a client that the resource they asked
97  // for has permanently moved to a new location. The response should
98  // also include this location. It tells the client to use the new URL
99  // the next time it wants to fetch the same resource.
100  "302 Found",
101  // 302 tells a client that the resource they asked for has temporarily
102  // moved to a new location. The response should also include this location.
103  // It tells the client that it should carry on using the same URL
104  // to access this resource.
105  "304 Not Modified",
106  // 304 status code is sent in response to a request (for a document)
107  // that asked for the document only if it was newer than the one the
108  // client already had. Normally, when a document is cached, the date
109  // it was cached is stored. The next time the document is viewed, the
110  // client asks the server if the document has changed. If not, the
111  // client just reloads the document from the cache.

120  "403 Forbidden",
